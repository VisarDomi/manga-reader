Frontend Restructure Direction

The current frontend should not be patched with more gates. The stronger structure is a small set of command owners:

  • ScrollOwner: the only code allowed to write DOM scroll positions.
  • ProjectionOwner: owns logical-to-physical window projection and rebase eligibility.
  • ProgressOwner: accepts open/visible/close commands and serializes progress writes.
  • RootRestoreOwner: already partly exists; should fully own root replay and make AppState thinner.
  • ImageVisibilityOwner: owns critical vs preload image loading and visibility guarantees.
